# DeepOrbit Vault Structure Template

This document defines the strict, canonical folder hierarchy for the DeepOrbit knowledge base.
The Vault Custodian AI Agent uses this template as the single source of truth when running the `/do:organize` skill.

## 0. Root Directory Cleanliness
No Markdown files or media should exist in the root directory except system configuration files (e.g., `DeepOrbitPrompt.md`, `deeporbit.json`). All content must be housed within one of the numbered folders below.

## 1. Numbered Folders Taxonomy

### `00_Inbox/`
Unprocessed notes, quick captures, ideas, and fleeting thoughts. Content here should be reviewed and moved elsewhere.

### `10_Diary/`
Daily logs and journals. Format: `YYYY-MM-DD.md`. Includes morning plans, daily recaps, and news summaries.

### `15_Writings/`
Drafts, essays, and polished personal writing produced through `do.write`.

### `20_Projects/`
Active projects with a defined start and end. Each project should have its own dedicated subfolder to house related materials and plans. Completed projects should be moved to `99_System/Archive/`.

### `30_Research/`
Deep dive research and exploration notes on complex topics. Often contains literature reviews and long-form analysis.

### `40_Wiki/`
Atomic concepts, definitions, and evergreen structured knowledge. Information here should be interconnected via wikilinks.

### `50_Resources/`
External materials, newsletters, product manuals, clippings, and source documents. Do not write original insights here.

### `60_Notes/`
Summaries, reading notes, meeting minutes, and book highlights.

### `90_Plans/`
High-level execution plans, strategy documents, or step-by-step AI workflows generated by `/do:kickoff`.

### `99_System/`
System configuration, templates, scripts, embedded attachments, and archives.
- `99_System/Templates/`: Templates for creating new types of notes.
- `99_System/Bases/`: Native Obsidian Bases for projects, research, task sources, and knowledge health.
- `99_System/Calendar/`: Portable calendar exports such as `DeepOrbit.ics`.
- `99_System/Archive/`: Archived projects and legacy notes no longer needed but kept for reference.

## 2. Structural Principles
- **Keep it Flat**: Avoid nesting folders deeper than 2 levels whenever possible (e.g., `40_Wiki/AI/` is acceptable, but `40_Wiki/Tech/Software/AI/` is discouraged).
- **Semantics over Granularity**: Folders should represent broad domains. Use `#tags` and Maps of Content (`[[MOC]]`) to group highly specific topics instead of creating endless subfolders.
- **No Empty Shells**: Folders with no files should be purged.
- **No Orphans**: Unlinked files in `00_Inbox` are acceptable; however, notes in `30_Research` or `40_Wiki` should eventually link to concepts or MOCs.
