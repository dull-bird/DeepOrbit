# User Profile

## 近期状态（草稿）
- 已拿到华为 Offer 并即将入职。
- 当前处于离职过渡期，需要处理交接。
- 对前同事与既有关系有明显情绪负担，担心在离场后出现重复接触。
- 偏好在分析与决策场景下采用理性求真模式；日常对话可放松。

## 核心价值与决策偏好
- 第一原则：把结论建立在事实和可验证证据上。
- 风险意识：在结论前先列边界与反例。
- 执行导向：每次分析尽量给出可执行下一步。
- 结构化表达：喜欢拆解、排序、比较、复盘。

## 认知与沟通偏好（可调用的分析模式）
1. 事实优先于情绪表达。
2. 允许我给出不确定性与信息缺口，而不是虚构“确定答案”。
3. 长链推演时保持步骤清晰。
4. 适度可视化（流程图/思维导图/象限图/XY 图）。
5. 对客观知识类问题给出可追问、可验证的建议。
6. 结尾可以给 1-2 个苏格拉底式反问，帮助我反向检验判断。

## 分析口径（草稿）
### 目标
- 把焦虑从“反复猜测”切换到“可执行动作”。
- 在入职过渡期稳定情绪与行为边界。
- 避免把离职人际冲突扩大为泛化认知负担。

### 关注指标
- 睡眠/焦虑波动（主观 1-10）。
- 交接任务完成率（已完成/待办）。
- 关键沟通是否一次性说明清楚（是/否）。
- 每日一次“客观复盘”：今天证据、判断、下一步。

## Mermaid 规则（可直接作为分析约束）
- Mermaid 图内中文文本可用。
- 代码块内禁止全角标点。
- 流程图优先：`flowchart TD`。
- 条件连线必须写：`A -->|条件| B`。
- 思维导图纯缩进，不使用 `-`/`*` 列表符。
- 象限图使用英文标签。
- XY 图使用 `xychart-beta`，明确 `x-axis` 与 `y-axis`。
